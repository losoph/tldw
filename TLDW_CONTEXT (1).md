# TLDW — Контекст проекта для Claude Code

_Файл контекста воркспейса. Обновлён: июль 2026._
_Продукт: self-hosted сервис саммаризации аудио/видео/документов для малой команды (3–7 человек, Росатом-контур)._

---

## 1. Ключевой принцип продукта

**Саммари — это ядро и основная задача приложения.** Все остальные возможности (строгий режим, privacy/локальная модель, транскрипт в выдаче, таймкоды, ссылки) — это **модификаторы поверх саммаризации**, они её настраивают, но никогда не выключают. Саммари может быть модульным (собирается из секций), но присутствует в выдаче всегда.

Единственное исключение из видео-центричности: **вкладка «Документы» (alpha)** — саммаризация офисных файлов и переписок, отдельный интерфейс, тот же LLM-контур.

## 2. Текущее состояние (v1.0 в продакшне)

Развёрнуто на VPS VDSina (2 vCPU / 4 ГБ / Ubuntu 24.04), переезд запланирован (см. §5, v1.1).

**Работает:**
- Загрузка видео/аудио до 2 ГБ (drag&drop): mp4, webm, avi, mov, mkv, m4v, mp3, wav, ogg, m4a, flac, aac, opus, wma
- Пайплайн: FFmpeg (аудио 16кГц WAV) → faster-whisper (small, int8, CPU) → DeepSeek API → структурированное саммари
- Очередь на SQLite, один воркер, поллинг статусов, история задач с удалением
- Оценка времени по RTF (real-time factor, калибруется по логам, env `RTF=2.0`)
- UI: тёмная тема, брендинг «TLDW — саммари ваших аудио и видеофайлов», таймер прошло/осталось
- nginx: auth_basic, лимит 2 ГБ, server_tokens off; Fail2ban; ufw (22/80/443); systemd-сервис `tldw.service`; CLI `tldw` (start/stop/rebuild/logs/status)
- Авто-сброс зависших задач при рестарте воркера
- Проект в GitHub (публичный), секреты в .gitignore (.env, nginx/.htpasswd, data/)

**Стек:** Python 3.11, FastAPI, uvicorn, faster-whisper 1.0.3, openai 1.35.0 (+пины httpx==0.27.2, requests==2.32.3), FFmpeg/ffprobe, SQLite, Docker Compose, nginx:alpine.

**Параметры Whisper (worker.py):** model=small, compute_type=int8, cpu_threads=4, beam_size=2, vad_filter=True (min_silence_duration_ms=500), condition_on_previous_text=False, initial_prompt="Транскрипция вебинара на русском языке.", language=ru.

**Структура проекта:**
```
whisper-app/
├── app/
│   ├── main.py          # FastAPI: /upload, /status/{id}, /jobs, DELETE /jobs/{id}
│   ├── worker.py        # цикл: pending → ffmpeg → whisper → deepseek → done
│   ├── requirements.txt
│   ├── Dockerfile       # python:3.11-slim + ffmpeg
│   └── templates/index.html
├── nginx/nginx.conf     # + .htpasswd (не в git)
├── docker-compose.yml   # nginx, web, worker; volumes: data/, whisper_cache, templates
├── .env                 # DEEPSEEK_API_KEY, WHISPER_MODEL, LANGUAGE, RTF (не в git)
├── tldw.sh, tldw.service
└── docs/ (DEPLOY, ARCHITECTURE, OPERATIONS, TROUBLESHOOTING, ROADMAP, GITHUB)
```

**БД jobs:** id, filename, status (pending/extracting_audio/transcribing/summarizing/done/error), created_at, transcript, summary, error, audio_duration, started_at, estimated_seconds.

**Известные грабли (не повторять):**
- `sed` по worker.py ломает отступы → только полная перезапись файлов
- Терминал VPS не принимает вставку >5 КБ → большие файлы через scp
- После добавления volume в docker-compose нужен `down && up`, не `restart`
- python:slim не имеет curl → healthcheck через python urllib
- openai==1.35.0 конфликтует со свежим httpx → пин httpx==0.27.2
- PasswordAuthentication=yes возвращается политикой VDSina → принято, компенсировано Fail2ban

## 3. Матрица режимов (проектная основа)

### Видео/аудио

| Публичность ↓ / Режим → | Саммари (основной) | + Строгий (стенограмма) |
|---|---|---|
| **Публичный** | DeepSeek; модульное саммари; транскрипт по галочке | ✅ DeepSeek: Whisper medium+beam=5+глоссарий, дословные цитаты с таймкодами, секция сомнений |
| **Непубличный (privacy)** | Qwen3-4B локально; модульное саммари; файлы до ~20 мин; транскрипт по галочке доступен | ❌ Недоступен до апгрейда RAM (16 ГБ → large-v3 + Qwen3-14B). UI объясняет: «privacy → без выверенных цитат» |

Правила:
- Строгий режим требует явного подтверждения «материал публичный» (радиокнопка, без автоматики)
- Privacy + длинный файл → UI предлагает: сократить таймкодами / пресет «Экспресс» / отказаться
- Саммари не выключается никогда; строгий режим ДОБАВЛЯЕТ выверенную стенограмму к саммари

### Документы (вкладка alpha) — privacy инвертирован!

| По умолчанию | Локальная модель (Qwen3-4B) — документы считаются служебными |
|---|---|
| «Быстрый» режим (DeepSeek) | Активируется только после явного подтверждения пользователем: «все загруженные файлы не ограничены в доступе» |

### Пресеты модульного саммари (подход А: один вызов LLM, промпт собирается в бэкенде из секций; промпты пользователю НЕ показываются и им не редактируются)

Видео: **Полное** (все секции — дефолт) / **Экспресс** (суть + action items) / **Конспект** (суть + конспект + навигация по таймкодам).
Секции видео: суть, конспект по блокам с таймкодами, инсайты, action items, Q&A, глоссарий, навигация.

Документы (черновик, скорректировать после альфы):
- **Документ** (docx/pdf/md/txt/rtf/html): суть → ключевые положения → цифры и факты → выводы → action items
- **Данные** (xlsx): что за данные → ключевые показатели → аномалии/выбросы
- **Переписка** (json = Telegram-экспорт, определяется по расширению): участники и тема → хронология решений → договорённости → открытые вопросы

## 4. Спецификация: вкладка «Документы» (alpha)

**Концепция UI:** отдельная вкладка со своим интерфейсом. Три категории контента: **документы / данные / чаты** (разделение по типу файла автоматически).

**Форматы:** docx, md, pdf (только текстовый слой, БЕЗ OCR — на скан честный отказ «распознавание не поддерживается»), xlsx, pptx, txt, rtf, html, json (только Telegram-экспорт → режим «Переписка»). Старые doc/xls НЕ поддерживаем — просим пересохранить.

**Парсеры:** python-docx, python-pptx, openpyxl, pymupdf, striprtf, BeautifulSoup; для Telegram JSON — свой препроцессор: `JSON → "ДД.ММ ЧЧ:ММ Автор: текст"` (компактный сценарий диалога, реплаи/пересылки сворачиваются).

**Пайплайн:** файл → parsing (секунды) → LLM → саммари. Переиспользуются: воркер, очередь, статусы, история. Новый статус `parsing` вместо extracting_audio/transcribing. Время обработки через DeepSeek ~1–3 мин.

**Лимиты:** документы ≤ 5 МБ; PDF ≤ 20 МБ; переписка ≤ 10 МБ на файл; **сессия: до 8 файлов, суммарно < 50 МБ** (лимит единый для обоих режимов). Очень длинные тексты (>~90k токенов) — обрезка с честной пометкой «обработаны первые N страниц».

**Загрузка и сессии:** по 1 файлу за раз, НО саммари можно запросить по нескольким загруженным файлам сразу. Сессия: **до 8 файлов, суммарно < 50 МБ** (единый лимит для обоих режимов). Схема map-reduce: каждый файл автоматически получает краткое саммари сразу после загрузки (map, конвейеризуется), кнопка «Сформировать общее саммари» — финальная сборка (reduce). В privacy reduce работает по промежуточным выжимкам (map-выжимки ≤800 ток.), в быстром режиме — по полным текстам (качество выше). Механику отображения сессии в UI/истории проработать (см. открытые вопросы).

**Privacy:** локальный режим по умолчанию; «быстрый» (DeepSeek) — после подтверждения открытости всех файлов.

## 6. Зафиксированные решения

| Решение | Обоснование |
|---|---|
| Саммари — ядро; строгий/privacy — модификаторы, не выключающие его | принцип продукта |
| Гибрид: DeepSeek для обычных, Qwen локально для приватных | скорость+качество API, приватность локально |
| Новый VPS 4c/8ГБ/50ГБ SSD вместо 6c/16ГБ/80ГБ NVMe | экономия ~6.5к ₽/мес; privacy пока тестовый; диск не bottleneck (Whisper/LLM compute-bound) |
| Ubuntu 24.04 LTS | стек и документация обкатаны; экономии на другой ОС нет |
| Qwen3-4B для тестов → 8B/14B после апгрейда RAM | лучшая открытая линейка для русского |
| Подход А для модульного саммари: один вызов, сборка промпта в бэкенде | Qwen не потянет мультивызовы (prefill повторяется); промпты не выносятся на пользователя |
| Транскрипт в выдаче — по галочке, бесплатен (Whisper всё равно его делает) | нулевая цена в токенах |
| yt-dlp для VK/VK Video/YouTube; только аудио | один инструмент, ~50–100 МБ/час вместо гигабайтов |
| Прокси per-domain конфигом; YouTube с рос. IP нестабилен, VK напрямую | сознательно выбран российский IP для скорости загрузки |
| Корпоративный VK-аккаунт + cookies для видео «по ссылке» | только свои/росатомовские ролики — правовой вопрос закрыт |
| Строгий режим: только публичные материалы, только DeepSeek (до апгрейда) | Qwen3-4B слаба для выверки; privacy→«без цитат» прописан в UI |
| Пре-чек качества: уровень 1 всем, уровень 2 строгому режиму | не замедлять массовый сценарий |
| Документы: старые doc/xls не поддерживаем; PDF без OCR | пересохранение — дешевле поддержки legacy |
| Документы: privacy по умолчанию, DeepSeek по явному подтверждению | инверсия относительно видео — служебные файлы по дефолту закрыты |
| json = только Telegram-экспорт, включает режим «Переписка» | по расширению, других json не ждём |
| Яндекс 360 = документы+бэкапы (rclone sync), VPS = БД+индексы | WebDAV ломает SQLite/индексы |
| Лимиты документов: 5 МБ / PDF 20 / чат 10 / всего 50 | alpha-рамки |
| Map-reduce для сессий документов: авто-саммари каждого файла (не опция) + «Общее саммари» по запросу | обходит контекст Qwen3-4B; конвейеризация; саммари = ядро продукта; в быстром режиме reduce из полных текстов |

## 7. Открытые вопросы и незакрытые задачи

**Документы (alpha) — требуют решения до/в ходе реализации:**
1. **Сессии загрузок:** как ограничивать состав общего саммари — все файлы за X часов? явная «корзина» с кнопкой «саммаризировать выбранное»? как сессия отображается в истории?
2. ~~Сводное саммари в privacy~~ **РЕШЕНО (map-reduce):** каждый документ автоматически получает краткое саммари сразу после загрузки (map; не опция — это ядро продукта + конвейеризация с загрузкой следующих файлов); кнопка «Сформировать общее саммари» собирает финал (reduce). Требования: map-промпт сжимает БЕЗ потери фактуры (цифры, фамилии, даты, формулировки решений — сохранять принудительно); ветвление по режиму — privacy: reduce из промежуточных выжимок; быстрый режим: общее саммари из ПОЛНЫХ текстов (контекст DeepSeek вмещает, качество выше), промежуточные выжимки остаются как самостоятельная ценность; лимит сессии — до 8 файлов и суммарно < 50 МБ (единый для обоих режимов); в privacy дополнительно map-выжимки ≤800 токенов, чтобы reduce гарантированно влезал в контекст 4B.
3. **xlsx-парсинг:** сколько листов/строк брать, как сериализовать таблицу в текст для LLM (CSV-фрагмент? описание колонок + сэмпл?). Прототипировать.
4. **Пресеты документов** — черновик из §3 скорректировать после первых реальных прогонов на служебных файлах.
5. **Telegram-препроцессор:** что делать с медиа-сообщениями (фото/голосовые = «[фото]»?), с очень длинными чатами (лимит 10 МБ ≈ сотни тысяч сообщений — нужна ли доп. обрезка по датам?).
6. Нужна ли история/выдача транскрипта аналога для документов (полный извлечённый текст в выдаче)?

**Видео-контур:**
7. **Прокси для YouTube:** источник прокси не выбран (свой второй VPS за рубежом? покупной?). Формат proxies.yml согласован концептуально, не спроектирован.
8. **Процедура обновления VK-cookies:** кто и как переэкспортирует (инструкция для команды), срок жизни неизвестен — выяснить эмпирически.
9. **Глоссарий строгого режима:** кто пополняет glossary.txt, где живёт (в git? в data/?).
10. **Пороги пре-чека** (🟢🟡🔴 по avg_logprob и метрикам) — калибровать на реальных записях.
