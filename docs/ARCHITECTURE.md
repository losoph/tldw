## Поток обработки

1. **Загрузка** — файл (multipart-POST на `/upload`) или ссылка VK/YouTube (JSON на `/upload_url`); в обоих случаях передаются опции: пресет саммари, список фрагментов «с–по» (можно несколько из одного видео), галочка «включить полную стенограмму в саммари»
2. **Приём** — nginx буферизует и проксирует на FastAPI (`web`)
3. **Сохранение** — `web` сохраняет файл в `/data/uploads/{uuid}.{ext}` (или только URL), создаёт запись в SQLite со статусом `pending`
4. **Поллинг** — каждые 3 секунды `worker` смотрит в БД на задачи в статусе `pending`
5. **yt-dlp** (только для ссылок) — скачивание всей аудиодорожки → статус `downloading`; каскад попыток (с cookies / без cookies / разные `player_client` для YouTube), прокси per-domain из `data/proxies.yml`, cookies из `data/cookies/<домен>.txt` (подробности — [VIDEO_LINKS.md](VIDEO_LINKS.md))
6. **FFmpeg** — извлечение аудио в WAV 16kHz mono → статус `extracting_audio`; каждый фрагмент режется отдельным `-ss`/`-to` (перед `-i`) в свой WAV
7. **ffprobe** — замер длительности, расчёт оценки времени
8. **Пре-чек качества (уровень 1)** — битрейт, средняя громкость, доля тишины (~секунды); результат сохраняется в `precheck`, 🟡/🔴 показываются баннером в UI, обработка не блокируется
9. **Whisper** — транскрибация локально → статус `transcribing`; каждый фрагмент транскрибируется отдельно, таймкоды смещаются к исходнику, части склеиваются (при нескольких фрагментах — с заголовками)
10. **DeepSeek** — промпт собирается в бэкенде из секций пресета (Полное / Экспресс / Конспект), один вызов LLM → статус `summarizing`
11. **Завершение** — сохранение саммари и транскрипта в БД → статус `done`. Все временные файлы (исходник, WAV, `.title`, `.part`) удаляются в `finally` — и при успехе, и при ошибке
12. **Отображение** — фронтенд поллит `/status/{id}` каждые 1.5 сек и обновляет UI

## Структура данных

### SQLite таблица `jobs`

| Поле | Тип | Описание |
|---|---|---|
| `id` | TEXT PK | UUID задачи |
| `filename` | TEXT | оригинальное имя файла |
| `status` | TEXT | pending / downloading / extracting_audio / transcribing / summarizing / done / error |
| `created_at` | REAL | unix timestamp создания |
| `summary` | TEXT | финальное саммари от DeepSeek |
| `error` | TEXT | сообщение об ошибке если status='error' |
| `audio_duration` | REAL | длительность аудио в секундах |
| `started_at` | REAL | unix timestamp начала обработки |
| `estimated_seconds` | INTEGER | оценка времени = audio_duration × RTF |
| `transcript` | TEXT | транскрипт с таймкодами (сохраняется всегда; выдаётся в UI по галочке) |
| `clip_start` / `clip_end` | REAL | первый фрагмент (legacy/для отображения); полный список — в `clips` |
| `clips` | TEXT | JSON-список фрагментов `[[start,end], ...]` в секундах (пусто/NULL = целиком) |
| `source_url` | TEXT | исходная ссылка для задач «по ссылке» (NULL = загрузка файлом) |
| `include_transcript` | INTEGER | 1 = включить полную стенограмму в выдачу |
| `sections` | TEXT | JSON-список секций саммари (из пресета; шаблоны — в worker.py) |
| `precheck` | TEXT | JSON результата пре-чека: level (green/yellow/red), messages, metrics |
| `owner` | TEXT | логин пользователя (из nginx basic-auth, заголовок X-Auth-User); история изолируется по owner |

### Файловая система (volume `data/`)

```
data/
├── jobs.db           # SQLite база
├── uploads/          # временные исходные файлы (удаляются после обработки)
├── audio/            # временные WAV для Whisper (удаляются после обработки)
├── cookies/          # netscape-cookies для yt-dlp: <домен>.txt (например vk.com.txt)
├── bin/deno          # JS-рантайм для YouTube n-challenge (linux x86_64, вне образа)
└── proxies.yml       # прокси per-domain для yt-dlp (см. proxies.yml.example)
```

### Volume `whisper_cache`

Хранит скачанные модели faster-whisper в `/root/.cache/huggingface/`. Сохраняется между перезапусками контейнеров — модель скачивается один раз.

## Безопасность

- **nginx** на входе: `auth_basic` блокирует анонимный доступ. Файл `.htpasswd`
  генерируется entrypoint-скриптом контейнера из `WEB_LOGIN`/`WEB_PASSWORD` и
  `TLDW1..4_PASSWORD` (`.env`) при каждом старте — секреты не хранятся в репозитории
- **Изоляция истории по пользователю**: nginx пробрасывает логин basic-auth в web
  заголовком `X-Auth-User` (`$remote_user`); web пишет его в колонку `owner` и фильтрует
  `/jobs`, `/status`, `DELETE` по нему. Порт 8000 доступен только внутри docker-сети,
  поэтому клиент не может подделать заголовок
- **Ротация логов**: json-file docker-логи ограничены 10 МБ × 3 файла на контейнер
- **`server_tokens off`** скрывает версию nginx
- **fail2ban** на SSH (если настроен) банит IP после 4 попыток
- **Docker network isolation**: порт `8000` (FastAPI) виден только внутри docker network, наружу торчит только nginx :80
- **healthcheck**: контейнер `web` периодически проверяется, при падении docker автоматически рестартит
- **`restart: unless-stopped`** на всех контейнерах: после reboot VPS поднимаются автоматически

## Ограничения текущей реализации

- **Один worker** — задачи обрабатываются последовательно. При нескольких пользователях очередь может расти.
- **SQLite-поллинг** — не подходит для распределённой обработки. Для масштабирования нужен Celery + Redis.
- **HTTP без TLS** — пароль передаётся в base64. Для продакшна нужен HTTPS через Let's Encrypt (требует домен).
- **CPU-only** — для GPU нужно изменить `device="cuda"` в worker.py и базовый образ Docker.

## RTF и оценка времени

Real-Time Factor — отношение времени обработки к длительности аудио.

| Конфигурация | RTF (примерно) |
|---|---|
| small + beam=2 + VAD на 2-core CPU | ~2.0 |
| medium + beam=2 + VAD на 2-core CPU | ~4.5 |
| small на 4-core CPU | ~1.2 |
| any на GPU | ~0.1 |

После каждой задачи воркер логирует фактический RTF — используйте его для калибровки `RTF` в `.env`.
